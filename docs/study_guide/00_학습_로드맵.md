> KAIST 학부 3학년 기준, 바이브 코딩으로 작성된 NPU 프로젝트를 한 줄 한 줄 이해하기 위한 학습 가이드

## 1. 프로젝트 한눈에 보기

이 프로젝트는 **PyTorch 모델 → NPU 하드웨어 실행**까지의 풀스택을 구현합니다.

```
┌─────────────────────────────────────────────────────────────┐
│                    소프트웨어 (L6 Toolchain)                   │
│                                                               │
│  PyTorch 모델 → tracer → IR → lowering → scheduler           │
│                → emitter → workload package (JSON)            │
│                → replay_bridge → RTL 시뮬레이션 검증           │
└──────────────────────────┬──────────────────────────────────┘
                           │ JSON manifest + activation/weight/golden
                           ▼
┌─────────────────────────────────────────────────────────────┐
│                    하드웨어 (RTL)                              │
│                                                               │
│  AXI MMIO (host) → System FSM → DMA → SRAM (ping-pong)       │
│                  → Core (skew → 16×16 systolic array → acc)   │
│                  → drain output → golden 비교                  │
└─────────────────────────────────────────────────────────────┘
```

### 핵심 개념 3가지

| 개념 | 설명 |
|------|------|
| **Weight-Stationary** | Weight를 PE에 한번 로드하면 고정. Activation만 흘려보내며 MAC 연산 |
| **Systolic Array** | 16×16 = 256개 PE가 격자로 연결. 데이터가 파도처럼 흘러가는 구조 |
| **Tiled Execution** | 큰 행렬을 16×16 타일로 잘라서 반복 실행. Split-K로 K축도 분할 |

---

## 2. 전체 데이터 플로우 (7단계)

```
① DMA가 외부 메모리에서 Weight 데이터를 읽어 SRAM Bank에 저장
② Bank Swap: DMA가 쓰는 뱅크 ↔ Core가 읽는 뱅크 교대
③ Weight Preload: SRAM에서 256개 PE로 weight 값을 로드 (16 cycle)
④ DMA가 Activation 데이터를 읽어 SRAM Bank에 저장
⑤ Bank Swap 후, Activation Execute: skew → MAC → deskew (31 cycle)
⑥ Accumulator에 partial sum 누적 (Split-K 시 여러 타일 결과 합산)
⑦ Drain: 최종 결과를 accumulator에서 추출 → golden과 비교
```

## 3. 프로젝트 구성 요소 (6개 영역)

### 영역 A: RTL 하드웨어 (19 파일)
NPU 칩의 실제 회로를 SystemVerilog로 기술

### 영역 B: 테스트벤치 (10 파일)
하드웨어를 검증하는 시뮬레이션 환경 (Directed Test + UVM + Verilator C++)

### 영역 C: L6 컴파일러 툴체인 (19 파일)
PyTorch 모델 → NPU 실행 패키지 변환 Python 파이프라인

### 영역 D: 호스트 드라이버 (6 파일)
Verilator 시뮬레이션 및 FPGA 보드 연동을 위한 C++/Jupyter 인터페이스

### 영역 E: 빌드/스크립트 (16 파일)
시뮬레이션 실행, 검증, CI를 위한 자동화 스크립트

### 영역 F: FPGA 구현 (4+3 파일)
Vivado 합성 스크립트 + FPGA 전용 RTL 래퍼

