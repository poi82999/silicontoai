## 추천: 경로 A (RTL & 검증) ← 학부생이 가장 빠르게 임팩트 낼 수 있음

### 왜 A가 좋은가?

1. **즉시 기여 가능**: RTL 코드는 눈으로 읽기 쉽고, assertion/테스트를 추가하면 바로 "이 학부생이 버그를 찾았네" 평가 받음
2. **자체 프로젝트 수행 불필요**: 기존 RTL을 개선하는 방식이라 "큰 그림"에 집중할 필요 없음
3. **깊이 있음**: 논문 수준의 검증 기법 배울 수 있음 (coverage, formal verification)
4. **교수님과의 대화**: 버그를 찾으면 자연스럽게 office hour에서 토론 시작
5. **대학원 지원에 강함**: RTL 설계 경험 + UVM 경험이 있는 학부생은 매우 드묾

**Assertion 추가 (2주)**

찾은 문제들을 체계적으로 assertion으로 변환:

```sv
// 예시 1: DMA address should be within bounds
property valid_dma_addr;
  @(posedge clk) disable iff (~rst_n)
  s_axi_awaddr inside {[0 : `DMA_ADDR_RANGE-1]};
endproperty
assert property(valid_dma_addr) 
  else $error("DMA address out of range!");

// 예시 2: SRAM port A와 B가 동시에 같은 address access 금지
property no_sram_port_conflict;
  @(posedge clk) disable iff (~rst_n)
  ~(port_a_en & port_b_en & (port_a_addr == port_b_addr));
endproperty

// 예시 3: Result valid는 DMA done 이후
property result_after_dma;
  @(posedge clk) disable iff (~rst_n)
  $rose(result_valid) |-> s_cycles_since_dma_done > 5;
endproperty
```

### 3주차: Fusion Rule 추가 (실제 코드 기여!)

**미션: Batch Normalization Fusion**

```python
# 현재: conv2d → relu → batch_norm (3개 separate 연산)
# 목표: conv2d → relu → batch_norm을 하나의 fused op로

# l6/src/l6_toolchain/fusion.py 에 추가:
class FusedConvReluBatchNormRule:
    """
    Pattern: conv2d → relu → batch_norm
    Fuses into: fused_conv_relu_bn
    """
    
    def match(self, graph):
        # graph에서 이 패턴 찾기
        ...
    
    def apply(self, matched_nodes):
        # 3개 노드를 1개로 합치기
        ...
```

**이게 왜 좋은가?**
- 합칠 수 있는 이유: activation 간 dependency 없음
- Benefit: 메모리 접근 줄임 (intermediate activation 저장 안 함)
- 측정: cycle count reduction by X%
